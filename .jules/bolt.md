## 2024-05-24 - [Fixing Broken React.memo and Expensive In-Render Computations]
**Learning:** Found significant performance overhead in the GTab App.tsx due to expensive array operations (`activeCategories`, `uniqueProfiles`, `filteredShortcuts`) being recomputed during every render cycle (like when dragging/dropping or typing in the search bar). This was compounded by `React.memo` on `ShortcutCard` failing silently because inline functions (`onEdit={(s) => setEditingShortcut(s)}` and `onFolderClick`) and unstable drag handlers were breaking reference equality. A benchmark confirmed a ~99% speedup when memoizing these derivations.
**Action:** When extracting computed arrays or filtered views, always declare them at the top-level of the component using `useMemo` with appropriate dependencies instead of evaluating them inside a render helper function. Secure child component memoization by consistently wrapping event handlers in `useCallback` or passing stable `useState` setters directly instead of wrapping them in anonymous functions.
