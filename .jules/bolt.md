
## 2024-05-18 - Rules of Hooks and Stale Closures with Inline Callbacks
**Learning:** In App.tsx, attempting to optimize inline event handlers on child components using `useCallback` requires immense care to prevent stale closures, especially with empty dependency arrays `[]` on drag-and-drop callbacks which might rely on the closure of other variables, even if functional updates are used. Furthermore, dynamically attempting to wrap variables computed deep in a switch block (e.g. `renderWidgetContent`) with `useMemo` violates the React Rules of Hooks, necessitating moving these computations strictly to the top level of the component scope.
**Action:** When extracting nested inline computations to top-level hooks like `useMemo`, ensure the placement strictly precedes any conditional early returns or render loop mappings, and ensure proper dependencies are listed. Avoid wrapping inline functions with `useCallback` blindly without exhaustive dependency tracking, especially for features like DnD.
