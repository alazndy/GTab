## 2025-04-04 - [Performance] Memoize expensive array operations in App.tsx
**Learning:** In App.tsx, expensive array operations like mapping, filtering and generating unique elements from the shortcuts array were performed inside the `renderWidgetContent` helper. Since this helper is called frequently during the React render cycle, not memoizing these operations led to performance bottlenecks, especially since the array computations depended strictly on a few state variables (like `shortcuts`, `filterCategory`, `filterProfile`).
**Action:** Always extract expensive computations out of the main render body or render helpers and wrap them in `useMemo` hooks with correct dependency arrays. This ensures computations are only re-run when their explicit dependencies change, significantly improving rendering performance.
