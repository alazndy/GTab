## 2024-05-24 - React.memo Broken by Inline Callbacks
**Learning:** `ShortcutCard` uses `React.memo` to optimize rendering, but its performance benefits were completely negated in `App.tsx` because callbacks like `onEdit`, `onFolderClick`, and event handlers (`handleShortcutDragStart`, etc) were defined as inline arrow functions or inside the component body without `useCallback`. This caused their references to change on every single render, forcing all child `<ShortcutCard>` components to re-render constantly. Furthermore, complex mapping and filtering arrays (`activeCategories`, `filteredShortcuts`) computed directly in the render cycle dramatically increased the cost of these re-renders.
**Action:** When implementing `React.memo` on heavily-used components, verify that all non-primitive props (arrays, objects, callbacks) passed from the parent are properly memoized using `useMemo` or `useCallback`. Otherwise, the memoization effort is wasted.
